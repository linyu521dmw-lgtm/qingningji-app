# 青柠集项目说明

## 项目名称

青柠集 - 校园闲置交换 App 原型

## 项目定位

青柠集是一个面向校园场景的本地可运行闲置交换 App 原型。项目围绕学生在校内转让教材、数码、宿舍用品、设计工具等物品的需求，提供从浏览、筛选、发布、管理到联系卖家的完整基础流程。

当前项目用于课程展示、作品集说明和功能原型验证，暂未上线，也没有接入真实用户系统或线上数据库。

## 目标用户

- 有闲置物品想在校内转让的学生
- 想低价购买教材、工具、宿舍用品的学生
- 需要快速了解商品状态和交易地点的校园用户
- 产品设计、交互设计课程中需要展示完整 App 流程的项目使用者

## 核心问题

校园闲置交易常见问题是信息分散、商品状态不清楚、沟通入口不统一、交易地点不明确。青柠集希望用一个轻量 App 原型，把发布、浏览、收藏、状态管理和联系卖家集中到同一个流程里，让校园闲置物品更容易被发现和流转。

- 手机端 App 页面 `frontend/mobile.html`
- 商品列表展示
- 商品搜索
- 分类筛选
- 商品详情弹窗
- 发布商品
- 图片上传和前端压缩
- 编辑商品
- 删除商品

- 我的收藏
- 我的发布

## 前端结构

- `frontend/index.html`：电脑端展示页，用于浏览项目整体视觉和基础入口。
- `frontend/mobile.html`：手机端 App 原型主文件，包含首页、分类、发布、商品详情、我的、消息会话、聊天详情等主要交互。

手机端当前采用单 HTML 文件实现，页面内包含结构、样式和脚本，方便本地演示和快速迭代。

## 后端结构

- `backend/server.js`：Node.js 后端入口，使用原生 `http` 模块提供 API。
- `backend/data/products.json`：当前商品数据。
- `backend/data/demoProducts.json`：演示商品数据，用于重置。
- `backend/data/transactions.json`：商品状态变化产生的交易记录。
- `backend/package.json`：后端运行脚本。
- `backend/README.md`：后端接口与运行说明。

## 主要接口列表

- `GET /health`：后端健康检查。
- `GET /api/categories`：获取商品分类。
- `GET /api/products`：获取商品列表。
- `GET /api/products/:id`：获取单个商品详情。
- `POST /api/products`：新增商品。
- `PATCH /api/products/:id`：编辑商品基础信息。
- `PATCH /api/products/:id/status`：修改商品状态。
- `DELETE /api/products/:id`：删除商品。
- `GET /api/transactions`：获取交易记录。
- `POST /api/reset-demo-products`：恢复演示商品数据，并清空交易记录。

## 当前技术栈

- 前端：HTML、CSS、原生 JavaScript
- 后端：Node.js 原生 `http` 模块
- 数据：本地 JSON 文件
- 图片处理：前端 Canvas 压缩后转为 Base64 数据
- 本地状态：`localStorage` 用于收藏和消息会话保留
- 版本管理：Git

## 当前项目状态

当前项目是一个本地可运行的校园闲置交换 App 原型，已经具备较完整的核心用户流程。它适合用于作品集展示、课程汇报和继续做交互细化。

项目仍处于原型阶段，没有真实登录、权限系统、线上部署、真实支付或实时聊天能力。后端数据使用本地 JSON 文件模拟，消息会话主要保存在浏览器本地。
