# 青柠集 MVP 回归测试清单

用于每次前端或后端更新后快速检查线上核心功能是否正常。

## 1. 测试前准备

- 打开线上前端地址。
- 使用带版本参数的链接，避免浏览器缓存影响测试，例如：`?v=YYYYMMDD-HHMM`。
- 准备账号 A 和账号 B。
- 准备一张小于 2MB 的商品图片。
- 准备一张超过 2MB 的测试图片，或准备 base64 curl 测试数据。
- 打开 Supabase Dashboard，方便检查 `products`、`transactions`、`favorites`、`conversations`、`messages`、`profiles` 和 Storage。

## 2. 基础访问测试

- 前端页面能正常打开。
- 后端健康检查正常：`GET /health` 返回 `ok: true`。
- 商品列表接口正常：`GET /api/products` 返回商品数据。
- 首页商品卡片正常展示标题、价格、分类、地点、卖家和状态。

## 3. 登录与用户资料测试

- 未登录状态下，“我的”页面只显示登录 / 注册入口，不显示用户数据面板。
- 使用账号 A 登录成功。
- “我的”页面显示账号 A 的邮箱。
- “我的”页面显示昵称 `displayName`。
- 点击“修改昵称”，输入新昵称并保存成功。
- 刷新页面后昵称仍存在。
- 退出登录成功，用户资料和数据面板消失。
- 重新登录账号 A，昵称和邮箱仍正常显示。

## 4. 商品发布测试

- 未登录点击发布，应提示先登录，不能发布商品。
- 账号 A 登录后发布无图商品成功。
- 账号 A 登录后发布带图商品成功。
- Supabase `products` 表写入 `owner_id = 账号 A user.id`。
- Supabase `products` 表写入 `owner_email = 账号 A email`。
- 商品 `seller` 使用账号 A 的 `profiles.display_name`。
- 如果 profile 昵称为空，`seller` 使用邮箱前缀。
- 带图商品上传后，Supabase Storage 中有图片文件。
- 带图商品的 `products.image_url` 有值。
- 发布成功后，首页商品卡片显示当前用户昵称作为卖家。

## 5. 商品编辑 / 删除 / 状态测试

- 账号 A 可以编辑自己发布的商品。
- 编辑商品标题、价格、分类、地点后，列表和详情页同步更新。
- 编辑商品时，不能通过前端把 `seller` 改成别人。
- 账号 A 可以删除自己发布的商品。
- 删除后商品不再出现在首页和“我的发布”。
- 账号 A 可以把自己的商品状态改为“已预定”。
- 账号 A 可以把自己的商品状态改为“已出”。
- 状态修改后，Supabase `transactions` 有记录。
- `transactions.actor_id = 账号 A user.id`。
- `transactions.actor_email = 账号 A email`。

## 6. 权限测试

- 未登录 curl 发布商品应失败。
- 未登录 curl 编辑商品应失败。
- 未登录 curl 删除商品应失败。
- 未登录 curl 修改商品状态应失败。
- 账号 B 不能编辑账号 A 的商品，应返回 403。
- 账号 B 不能删除账号 A 的商品，应返回 403。
- 账号 B 不能修改账号 A 商品状态，应返回 403。
- 账号 B 可以发布自己的商品。
- 账号 B 发布的商品 `owner_id` 应为账号 B。

## 7. 收藏测试

- 未登录点击收藏，应提示先登录，不能写入收藏。
- 账号 A 登录后可以收藏商品。
- “我的收藏”中显示账号 A 收藏的商品。
- 刷新页面后，“我的收藏”仍存在。
- 账号 A 可以取消收藏。
- 取消收藏后，“我的收藏”不再显示该商品。
- 账号 B 登录后，看不到账号 A 的收藏。

## 8. 消息测试

- 未登录点击“联系卖家”，应提示先登录，不能进入聊天详情。
- 账号 B 登录后，点击账号 A 商品的“联系卖家”可以创建会话。
- 同一账号对同一商品重复点击“联系卖家”，应复用已有会话，不重复创建。
- 会话写入 Supabase `conversations`。
- 会话初始化消息写入 Supabase `messages`。
- 账号 B 可以发送消息。
- 发送后 `messages` 表新增记录。
- 刷新页面后，消息仍存在。
- 账号 A 登录后，可以在消息列表看到与账号 B 的会话。
- 已知问题：手机键盘弹出后聊天记录可能被顶走，暂不作为阻塞项。

## 9. 表单校验测试

- 商品标题为空时，发布失败并提示。
- 价格不是数字时，发布失败并提示。
- 价格小于 0 时，发布失败并提示。
- 价格大于 9999 时，发布失败并提示。
- 分类选择“运动”时，后端能归一化为“运动户外”。
- 图片超过约 2MB 时，前端或后端应拦截并提示。
- 昵称为空时，修改失败并提示。
- 昵称超过 20 个字时，修改失败并提示。

## 10. 安全测试

- 线上调用 `POST /api/reset-demo-products` 应返回 403。
- 前端点击重置演示数据时，应提示“线上版本不支持重置演示数据”或等价提示。
- 前端文件中不应出现 Supabase service role key。
- `frontend/config.js` 或线上注入配置中只能出现 Supabase anon key。
- 未带 `Authorization: Bearer <access_token>` 的写接口请求应失败。
