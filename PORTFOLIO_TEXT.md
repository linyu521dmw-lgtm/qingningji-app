# 青柠集作品集文案

## 作品集项目介绍

青柠集是一个校园闲置交换 App 本地可运行原型，围绕学生在校内转让教材、数码产品、宿舍用品和设计工具的真实场景展开。项目从商品浏览、搜索分类、发布编辑、收藏、状态管理、交易记录到联系卖家和消息聊天，搭建了一条较完整的校园二手交易体验链路。当前版本用于作品集和课程展示，重点呈现产品流程、信息架构和核心交互，不是已经上线的正式产品。

## 简历项目经历版本

设计并实现“青柠集”校园闲置交换 App 原型，面向校内学生的二手物品流转场景，完成从需求梳理、功能规划、页面结构到可运行交互原型的搭建。项目包含商品列表、搜索筛选、发布商品、图片压缩、编辑删除、收藏、状态管理、交易记录、联系卖家、消息会话和聊天详情等功能；前端使用 HTML/CSS/JavaScript 实现移动端界面，后端使用 Node.js 和本地 JSON 文件模拟数据接口，支持本地完整演示。

## 1 分钟演示讲解稿

这是我的校园闲置交换 App 原型“青柠集”。项目面向学生在校内处理闲置物品的场景，比如教材、宿舍用品、数码产品和设计工具。进入手机端页面后，可以看到商品列表，也可以通过搜索和分类快速找到需要的物品。点击商品可以查看详情，包括价格、分类、交易地点、卖家和当前状态。

如果我是卖家，可以发布商品，上传图片，并在之后编辑或删除。商品状态可以在“我的发布”里切换为在售、已预定或已出，状态变化会进入交易记录。作为买家，我可以收藏商品，也可以点击联系卖家生成消息会话。消息页里可以看到会话列表、最后一条消息和时间，进入后可以查看聊天详情并发送本地消息。

这个项目目前是本地可运行原型，后端用 Node.js 提供接口，数据用 JSON 文件模拟。它的重点不是商业上线，而是展示一个校园闲置交易产品从浏览、管理到沟通的完整体验流程。

## 功能亮点列表

- 手机端 App 原型覆盖首页、分类、消息、我的等核心 Tab。
- 商品浏览流程完整，支持列表、详情、搜索和分类筛选。
- 发布流程支持图片上传，并在前端压缩图片，降低本地数据体积。
- 商品支持编辑、删除和状态切换，覆盖卖家管理场景。
- 收藏、我的收藏、我的发布帮助用户管理关注和发布内容。
- 商品状态切换会生成交易记录，便于展示物品流转过程。
- 联系卖家后生成消息会话，消息列表和聊天详情形成基础沟通闭环。
- 会话列表展示最后消息、时间和未读状态，更接近真实 App 使用体验。
- 后端 API 和本地 JSON 数据支持本地完整运行，便于演示和迭代。
- 支持一键重置演示数据，方便课堂展示和作品集录屏。

## 项目价值总结

青柠集把校园闲置交易中常见的“找不到、状态不清楚、沟通分散”问题整合进一个轻量 App 原型中。它通过商品信息、状态、收藏、交易记录和消息沟通的组合，展示了一个校园二手平台从信息浏览到交易沟通的基础产品闭环。作为作品集项目，它能够体现产品设计中的场景理解、流程拆解、界面组织和可运行原型实现能力。
